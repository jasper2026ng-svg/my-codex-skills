# 工具书转技能：来源质量标准 v0.1

## 核心原则

一本书适合阅读，不等于适合转成 Skill；一本书适合提供认知启发，也不等于能够单独支撑稳定的执行流程。

Book-to-Skill 必须分别判断：

1. 书本身的质量；
2. 这本书是否适合转成目标 Skill；
3. 当前提取是否覆盖了书的核心，而不是只覆盖用户划线的局部；
4. 哪些内容来自书，哪些来自 Agent 综合，哪些来自用户经验与案例。

## 一、证据来源及其边界

| 来源 | 主要价值 | 不能证明什么 |
|---|---|---|
| 目录与章节结构 | 作者如何组织问题、覆盖哪些主题 | 不能证明具体论证正确 |
| 用户个人划线与笔记 | 用户真正关注、理解或质疑的内容 | 不能代表全书，也不保证理解准确 |
| 全书公共热门划线 | 多数读者注意到的高密度观点 | 热门不等于核心，更不等于正确 |
| 推荐书评 | 读者认为有价值的部分及应用方式 | 可能过度赞美、二手转述或由 AI 整理 |
| 差评与一般书评 | 翻译、结构、证据和可读性风险 | 个别差评不能直接否定全书 |
| 作者、出版社及外部权威来源 | 作者背景、版本与出版可信度 | 权威身份不能替代方法有效性 |
| Agent 综合 | 把分散观点转成流程、边界和输出 | 可能加入原书没有的推断，必须标注 |
| 用户真实案例 | 检验方法是否适合用户场景 | 单个案例不能证明跨场景稳定 |

任何单一来源都不能独立代表“书的本质”。

## 二、书籍质量评分

评分用于比较和暴露风险，不是假装得到科学精确结论。

| 维度 | 权重 | 核心问题 |
|---|---:|---|
| 作者与出版来源可信度 | 15 | 作者是否有相关经验，版本与出版来源是否可靠？ |
| 目标任务相关度 | 15 | 是否直接处理目标 Skill 的任务边界？ |
| 概念完整与内部一致性 | 15 | 核心概念是否清楚、前后一致、覆盖关键边界？ |
| 证据与论证质量 | 15 | 结论来自研究、长期实践、案例还是单纯故事？ |
| 方法显性程度 | 15 | 是否提供步骤、判断规则和决策节点？ |
| 边界与失败模式 | 10 | 是否说明何时无效、会产生什么副作用？ |
| 案例与可理解性 | 10 | 案例是否帮助理解，翻译和表达是否清楚？ |
| 读者反馈与版本风险 | 5 | 是否存在一致的翻译、结构或误读问题？ |

等级：

- A（85–100）：可以作为强主来源；
- B（70–84）：可以作为重要来源，但需要补充方法或证据；
- C（55–69）：只适合作为局部参考或认知模型；
- D（低于 55）：不应作为 Skill 主来源。

## 三、可转 Skill 性审查

分别检查以下六项：

1. 触发场景是否清楚；
2. 输入信息是否可以定义；
3. 是否存在可执行步骤或判断规则；
4. 输出是否可以被检查；
5. 是否有质量标准与失败条件；
6. 是否能通过真实案例验证。

结论分为：

- 强：可直接作为原子 Skill 的主要方法来源；
- 部分：只能提供核心概念，执行流程需要其他来源与 Agent 综合；
- 弱：更适合做学习卡、心智模型或检查原则，不应强行写成 SOP。

## 四、提取时必须标注的来源标签

- `书中明确`：目录、划线或可靠书摘能够直接支持；
- `Agent 综合`：为执行稳定性新增的结构、步骤或模板；
- `用户经验`：来自用户真实工作判断与反馈；
- `案例验证`：经过真实任务测试并有结果；
- `待验证`：目前合理，但证据不足。

## 五、停止与降级规则

- 书籍质量为 D：停止作为主来源；
- 书籍只有观点和故事，没有稳定步骤：降级为学习卡或心智模型；
- 个人划线很少：不得声称是“基于用户笔记完整提取”；
- 公共划线集中于金句：必须回到目录和相反书评检查遗漏；
- 原书与 Agent 综合无法区分：不得发布为成熟 Skill；
- 真实案例持续暴露边界错误：先修正 Human Learning Card，再修正 Skill；
- 无法获得足够书籍内容：明确覆盖边界，不得声称还原全书本质。
