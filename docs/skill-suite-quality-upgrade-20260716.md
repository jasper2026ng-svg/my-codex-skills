# Book-to-Skill 六项升级复审 2026-07-16

## 目标

第一次审计确认 `critical-questioning` 和 `pyramid-communication` 的书源与任务边界最匹配。本轮不降低标准，而是回到微信读书，为其余六个 Skill 寻找能补上明确缺口的主书或校准书，并把变化写入 Skill、来源审计和人的学习卡。

书籍质量与 Skill 质量仍分开评价。分数是审计判断，不是科学测量，也不代表用户已经内化。

## 升级结果

| Skill | 第一次质量 | 升级后质量 | 新来源结构 | 关键升级 | 仍需警惕 |
|---|---:|---:|---|---|---|
| `real-problem-discovery` | 64，C | 76，B | 《你的灯亮着吗》+《好战略，坏战略》+《系统之美》 | 增加事件、模式、结构、关键挑战的层级检查 | 不要把“更深”误当“更真” |
| `structured-problem-framing` | 75，B | 81，B+ | 麦肯锡主书 + 系统与战略校准 | 先选问题树、流程图、利益相关者图、指标树或系统图 | 防止为复杂而复杂 |
| `first-principles-reframing` | 62，C | 74，B- | 轻量书只作触发；《系统之美》主校准；《表象与本质》补类比边界 | 加入系统边界、反馈、延迟、适应和局部优化 | 仍不是成熟标准 SOP |
| `effective-manager-operating-system` | 75，B | 83，B+ | 格鲁夫 + 德鲁克 + 猴子管理窄工具 | 团队产出、管理杠杆、指标、会议、任务成熟度 | 用户尚未内化格鲁夫 |

| `decision-quality-review` | 60，C | 80，B+ | 德鲁克有效决策 +《非对称风险》+ 偏差校准书 | 问题类型、边界条件、异议、妥协、行动反馈、后果对称 | 方案表和预演失败仍是综合层 |

## 没有采用的候选书

### 《学会决策》

目录仍以概率偏差、参考点、损失厌恶、情绪和直觉为主，只有 8 个评分记录。它可以补充行为经济学案例，但不能替代完整决策过程，因此没有升级为主书。

### 《组织诊断》

虽然用户读完并留下 44 条划线，目录也很全面，但公开评分样本中差评多于好评，负面反馈集中在模型拼盘、逻辑浅和缺少原创整合。因此只保留为模型和证据清单，不把它作为权威诊断主书。

### 《元思考力》

主题覆盖证据、概率、图像思维和系统思考，但用户未读，公开评分只有 30 个，且推荐书评中存在明显的概念堆叠式总结。它没有解决任何一个 Skill 的独特来源缺口，因此本轮不采用。

## 当前成熟度判断

两项原本较强的 Skill 加上本轮升级后的三项，形成第一梯队：

- `critical-questioning`
- `pyramid-communication`
- `effective-manager-operating-system`
- `structured-problem-framing`
- `decision-quality-review`

这不等于“成熟”。它们只是书源匹配、边界和执行合同已达到较好的测试版本。

第二梯队：

- `hrbp-business-diagnosis`
- `real-problem-discovery`

它们的现实价值高，但诊断仍依赖场景证据和人的判断。

谨慎使用：

- `first-principles-reframing`

它适合挑战惯例，不适合被设置为每次思考都必须经过的步骤。

## 人和 Agent 的不同成熟度

Skill 质量提高，不等于用户已经掌握。

- 用户已深度阅读或留下较多笔记：麦肯锡结构化思维、德鲁克、系统之美、猴子管理、组织诊断；
- 用户部分阅读：你的灯亮着吗、好战略坏战略、业务为本、非对称风险；
- 用户尚未内化：格鲁夫、杨三角第 3 版、表象与本质、思考快与慢。

因此，Agent 可以先按更新后的流程执行；人的学习卡必须继续标注阅读覆盖，并通过反向讲解和迁移测试内化，不能把 Agent 的掌握程度冒充成人的掌握程度。

## 发布前验证

每个升级 Skill 仍需检查：

1. 触发条件是否阻止误调用；
2. 第一轮是否按停止规则停下；
3. 是否只用与问题匹配的模型和偏差；
4. 是否把书中明确内容、Agent 综合和用户经验分开；
5. 是否在三个不同真实场景中产生可复盘的改进。
