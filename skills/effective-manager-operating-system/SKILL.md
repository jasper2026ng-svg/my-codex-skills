# Effective Manager Operating System

Use this skill to turn scattered management work into a clear operating system: what the manager contributes, where time goes, which priorities matter, how decisions are made, how meetings create output, and how ownership stays with the right person.

Before using this skill in coaching mode, ask the user for a short initial judgment: "What do you think is the biggest management bottleneck here?" Then compare the user's judgment with the skill output. Skip this only when the user explicitly wants direct execution.

Use `structured-problem-framing` first if the situation is still vague. Use `critical-questioning` when a management diagnosis may be based on weak evidence. Use `pyramid-communication` after this skill to turn the recommendation into a boss update or team message.

Do not summarize the source books or quote long passages from them. Treat the book-derived material as method inspiration, and write in your own words.

## Workflow

1. Identify the management outcome.
   - State what the manager is responsible for improving: output, decision speed, team ownership, execution quality, communication flow, or talent development.
   - Separate activity from contribution. A busy manager is not necessarily an effective manager.

2. Audit time and attention.
   - List the manager's recurring meetings, reviews, approvals, interruptions, and follow-ups.
   - Mark each item as contribution, coordination, control, development, firefighting, or avoidable noise.
   - Identify what should be stopped, shortened, delegated, batched, or redesigned.

3. Clarify priorities.
   - Name the few outcomes that matter most in the current cycle.
   - Separate urgent, important, and legacy commitments.
   - Check whether the manager is protecting time for the most important contribution.

4. Diagnose meetings and management rhythm.
   - Classify each meeting by purpose: decision, synchronization, problem solving, review, coaching, or ritual.
   - For each recurring meeting, define owner, input, decision/output, action owner, deadline, and follow-up mechanism.
   - Remove or redesign meetings that have no clear output.

5. Diagnose decisions.
   - Identify which decisions are stuck, recurring, or repeatedly escalated.
   - For each decision, define whether it needs the manager's judgment, team input, a rule, or delegation.
   - Require dissent or alternative options for important decisions before committing.

6. Diagnose delegation and "monkeys."
   - Identify work that has bounced back to the manager.
   - For each "monkey," name the next action, current owner, required freedom level, next check-in time, and escalation rule.
   - Keep ownership with the person doing the work unless there is a real risk or capability gap.

7. Build the operating system.
   - Define a weekly rhythm: decision meetings, 1:1s, reviews, deep work, follow-up checks.
   - Define a monthly rhythm: priorities, performance review, talent review, process improvement.
   - Define rules for escalation, meeting creation, decision rights, and follow-up.

8. Produce a practical change plan.
   - Recommend 3 to 5 changes, not a total management makeover.
   - Include the smallest experiment for one or two weeks.
   - Make success signals observable.

## Quality Bar

A good output should:

- focus on managerial contribution rather than activity volume;
- distinguish bad meetings from missing decision/coordination mechanisms;
- protect the manager's scarce attention;
- keep ownership with the right person instead of pulling work back to the manager;
- turn vague advice into cadence, rules, owners, and follow-up;
- avoid moralizing about attitude before checking system design;
- fit HRBP, sales management, emerging-market execution, and cross-cultural team contexts when relevant.

## When Not To Use

Do not use this skill when:

- the user only needs a simple calendar cleanup;
- the main issue is individual performance discipline requiring HR/legal procedure;
- the problem is actually strategy, market fit, or compensation design rather than management rhythm;
- there is not enough authority to change meetings, decision rights, or delegation rules;
- the user wants a polished message before the management diagnosis is clear.
