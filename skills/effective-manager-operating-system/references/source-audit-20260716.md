# 来源审计：管理者运行机制

## 来源组合

- 《卓有成效的管理者》：阅读 99%，个人划线 60 条；质量 84/100，B。
- 《别让猴子跳回背上》：阅读 100%，个人划线 21 条；质量 67/100，C。
- 《格鲁夫给经理人的第一课》：阅读 0%，个人划线 0 条；书籍质量约 90/100，A，但不属于用户已内化来源。

## 组合判断

德鲁克稳定支撑贡献、时间、优先级、用人所长和决策原则；“猴子”方法只适合支撑下一步责任与跟进，不应泛化成把问题退回下属；格鲁夫可以支持产出、会议和管理杠杆，但当前主要来自目录和公共资料。

完整的周/月运行机制、会议分类、自由度和升级规则主要属于 Agent 综合，不是任一本书直接给出的单一 SOP。

## 可转 Skill 性

**部分到强。** 任务边界实用，但组合来源必须明确，且要防止过度设计。

## 来源分层

- 书中明确：贡献优先、记录时间、要事优先、有效决策、下一行动责任、管理产出与会议用途。
- Agent 综合：完整运行机制、周/月节奏、会议模板、自由度和升级规则。
- 用户经验：先检查会议系统变量，不先归因团队态度。
- 待验证：不同团队规模和文化下，运行机制是否可持续。

## 修正结论

保留 Skill，但增加两条边界：管理者即使委派下一行动仍保留最终管理责任；在把任务留给下属前必须检查能力、资源、权限和风险。

## 多书升级复审

本轮重新审视《格鲁夫给经理人的第一课》的目录、1850 个评分记录、推荐与负面书评，并用企鹅兰登官方书籍页核验作者与原版定位。其内容直接覆盖团队产出、管理杠杆、指标、会议、决策、规划、组织结构、任务成熟度、绩效、招聘和培训，和本 Skill 的任务边界高度匹配。

来源分工调整为：

- 格鲁夫：运行机制主干；
- 德鲁克：贡献、时间、优先级和有效性原则；
- “猴子管理”：下一行动责任与复查的窄工具。

组合来源质量判断提升为 **89/100，A**。当前 Skill 质量暂评为 **83/100，B+**。剩余主要风险不是书源不足，而是用户尚未阅读格鲁夫，且 Agent 可能设计过多指标和会议。默认仍只允许提出 3 至 5 项调整和一个短周期试验。
