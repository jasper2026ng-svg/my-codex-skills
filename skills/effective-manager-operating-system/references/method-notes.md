## Method Anchors

From effective management:

- effectiveness can be learned
- manage time before managing tasks
- focus on contribution
- build on strengths
- put first things first
- make decisions deliberately
- use disagreement and alternatives before important decisions

From monkey management:

- work returns to managers when ownership is unclear
- a "monkey" is the next action, not the whole problem
- every monkey needs owner, next step, freedom level, and follow-up time
- managers should avoid taking back work that belongs with team members

From high-output management:

- management output depends on team output
- meetings are tools with different purposes
- decisions and planning need operating rhythm

## Skill Boundary

This skill answers:

- How should a manager redesign time, meetings, decisions, and delegation?
- What recurring rhythm should a manager use?
- Which work should stay with the manager and which should remain delegated?
- How can meetings create decisions, outputs, and ownership?

This skill does not answer:

- What is the real business problem? Use `structured-problem-framing`.
- Is the diagnosis evidence-backed? Use `critical-questioning`.
- How should the recommendation be communicated? Use `pyramid-communication`.
- How should HR compliance or disciplinary procedure be handled? Use HR-specific policies or a future HRBP skill.

## Common Failure Modes

- Treating all meetings as waste instead of diagnosing their missing output.
- Treating team attitude as the cause before checking management system design.
- Taking back work from team members in the name of speed.
- Creating an elaborate cadence that the team cannot sustain.
- Measuring manager effectiveness by busyness rather than contribution.
- Recommending delegation without defining freedom level and follow-up.
