# 来源审计：结构化问题定义

## 主来源

- 《麦肯锡结构化战略思维：如何想清楚、说明白、做到位》
- 作者：周国元
- 微信读书：阅读进度 99%，个人划线 43 条，个人想法 0 条
- 目录 61 条，公共热门划线 20 条，公开书评 1786 条

## 书籍质量

**77/100，B。**

优点：任务相关度高，五步法、问题切分、MECE、假设与验证的结构较显性；作者具有咨询实践背景，出版社可靠。

风险：内容覆盖思考、沟通和交付，边界较宽；部分工具来自既有咨询方法；公开负面反馈集中在冗余、方法浅和“怎么切”不够具体。书能提供流程和工具，不替代领域知识。

## 可转 Skill 性

**强，但应窄化。** 最适合提取问题定义、替代切法、问题树、工作假设和验证路径。不应同时承担真问题发现、完整证据审计和最终方案设计。

## 来源分层

- 书中明确：定义问题、结构化分析、替代维度、MECE、假设与验证、逻辑陷阱。
- Agent 综合：固定输出模板、分支数量、与其他 Skill 的路由和停止条件。
- 用户经验：事实与假设区分、管理场景中的现实验证成本。
- 待验证：是否能在不同 HRBP、销售和运营问题中稳定选择正确切法。

## 修正结论

Skill 应在 `real-problem-discovery` 已形成工作定义后使用。第一轮停止在问题定义、拆解、假设和验证需求，不直接给完整方案。

## 多书升级复审

主书仍然是最适合支撑结构化定义、替代切法、假设和验证的来源。本轮没有因为找到名气更大的书就替换它，而是补了两个校准来源：

- 《系统之美》：用户阅读 99%、划线 7 条，670 个评分记录。用于识别反馈、延迟、非线性和循环因果，防止把动态系统硬切成静态分类树。
- 《好战略，坏战略》：用户阅读 30%、划线 7 条，1180 个评分记录。用于检查结构是否抓住关键挑战和资源聚焦，而不是只有形式完整。

升级后的来源组合质量判断为 **84/100，A-**。当前 Skill 质量暂评为 **81/100，B+**。主要提升是新增“先选择问题表示方式，再拆解”的门槛；主要待验证风险是 Agent 可能过度选择系统图，或为了展示复杂度而增加无用结构。
