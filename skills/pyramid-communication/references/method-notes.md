## Skill Boundary

This skill answers:

- What is the top answer?
- What supporting points should sit under it?
- Are the supports grouped by a consistent logic?
- What should the audience understand, decide, or do?
- How should the message, memo, update, or slide storyline be written?

This skill does not answer:

- What is the real problem? Use `structured-problem-framing`.
- Is the claim supported by good evidence? Use `critical-questioning`.
- Should we redesign from basic facts? Use `first-principles-reframing`.

## Collaboration With Existing Skills

Typical chain:

```text
structured-problem-framing
-> critical-questioning
-> pyramid-communication
```

Use this chain when a messy business issue needs to become a clear boss update, decision memo, or meeting readout.

Alternative chain:

```text
first-principles-reframing
-> critical-questioning
-> pyramid-communication
```

Use this when a redesigned solution needs to be explained and defended.

## Common Use Cases

- Write a concise boss update.
- Turn meeting notes into an action-oriented readout.
- Convert a diagnosis into a recommendation memo.
- Build a slide storyline.
- Rewrite a long message so the conclusion comes first.
- Structure HRBP recommendations for business leaders.
- Explain a sales or organization problem to a non-HR audience.

## Output Discipline

- Prefer short, concrete sentences.
- Make the top answer visible within the first few lines.
- Turn labels into claims.
- Keep supporting points at the same logical level.
- Use situation-complication-question-answer only when it helps the audience follow the context.
- For senior audiences, state the recommendation and ask early.
- For resistant audiences, add just enough context before the recommendation.

## Common Failure Modes

- Creating a pretty outline without a real answer.
- Using headings that name topics but do not summarize meaning.
- Mixing causes, actions, risks, and evidence at the same level.
- Including every detail because it is available.
- Over-polishing wording while the logic remains weak.
- Forgetting the final ask.
