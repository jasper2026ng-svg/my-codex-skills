# 来源审计：HRBP 业务诊断

## 来源组合

- 《业务为本》：阅读 25%，个人划线 18 条、个人想法 2 条；73/100，B。
- 《人力资源量化管理与数据分析》：阅读 99%，个人划线 11 条；58/100，C。
- 《人力资源管理新逻辑：人人都是HR》：阅读 90%，个人划线 5 条；约 55/100，C，公开样本极少。
- 《HRBP是这样炼成的之中级修炼》：阅读 0%，个人划线 0 条；约 56/100，C。
- 《组织能力的杨三角》第 3 版：阅读 3%，个人划线 0 条；约 83/100，B，但不属于用户已内化来源。

## 组合判断

《业务为本》能支撑从业务出发、业务翻译、HR 解决方案和业务所有权，但公开反馈也集中指出概念多、术多、实操细节不足。数据书适合作为指标参考，不足以支撑因果诊断。其他辅助书提供角色和组织能力视角，但证据覆盖不均。

## 可转 Skill 性

**部分。** 可以支撑诊断框架，不能保证自动得到正确原因，更不能直接从症状跳到 HR 项目。

## 来源分层

- 书中明确：从业务结果出发、业务翻译、HRBP 与业务共同负责、数据支持假设、直线经理承担人员管理责任。
- Agent 综合：人/角色/系统切分、诊断输出模板、证据门槛、干预优先级和 Skill 路由。
- 用户经验：优先可观察、可改变、可复盘的组织变量。
- 待验证：尼日利亚和西非销售组织场景中的跨文化适用性。

## 修正结论

第一轮只完成业务翻译、组织诊断假设和证据需求。只有用户接受当前诊断后，第二轮才设计 HRBP 干预。

## 多书升级复审

本轮重点审计了《组织诊断》和重新定位《组织能力的杨三角》：

- 《组织诊断》：用户阅读 99%、个人划线 44 条，目录覆盖六个盒子、杨三角、7S、BLM、问卷、访谈、汇报线、生命周期和组织行为。但只有 24 个评分记录，其中好评 6、一般 7、差评 11；负面反馈集中在模型拼盘、逻辑浅和缺少原创整合。它适合做模型目录和证据提示，不适合作为诊断权威。
- 《组织能力的杨三角》第 3 版：用户阅读 3%、0 划线，86 个评分记录中 75 个好评。其“战略需要什么组织能力—员工能力—员工思维模式—员工治理方式”与本 Skill 的任务高度匹配，但当前仍不是用户已内化来源。

来源层级调整后，组合质量判断为 **84/100，A-**，当前 Skill 质量暂评为 **78/100，B**。主要进步是用一条业务到组织能力的主线替代杂乱的人力资源模块清单；主要风险是杨三角可能过于概括，因此必须把每一维转成可观察行为和机制，并保留竞争性假设。
